Miksi tämä sovellus on tehty?

Julkishallinnon WordPress-multisite-ympäristön pienkehitys vaatii tikettien hallintaa, koodikatselmuksia, deploy-pipeline-seurantaa ja saavutettavuusauditointia. Tämä sovellus simuloi AI-avusteista kehittäjäagenttia, joka auttaa hallitsemaan kaikki nämä tehtävät yhden chat-käyttöliittymän kautta.

Näin pääset alkuun

  1. Sovellus avautuu Chat-näkymään, jossa taustalla näkyy simuloitu WordPress-sivusto ja kelluva chat-widget.
  2. Kokeile chat-widgetiä — valitse konteksti (virasto, sivu, osio) ja keskustele agentin kanssa.
  3. Siirry Tiketit-välilehteen nähdäksesi kehityspyynnöt ja bugiraportit.
  4. Tarkastele koodimuutoksia Koodimuutokset-näkymässä diff-katseluna.
  5. Seuraa julkaisuprosessia Pipeline-näkymässä.
  6. Tarkista saavutettavuus Auditointi-välilehdellä.

CI/CD-pipeline ja julkaisuprosessin seuranta.

Kehitysmetriikat ja porautuvat kaaviot.

  • Sovellus toimii kokonaan selaimessa ilman palvelinta.

Why was this app built?

Managing small-scale development for a government WordPress multisite environment requires ticket management, code reviews, deploy pipeline tracking and accessibility auditing. This app simulates an AI-assisted developer agent that helps manage all these tasks through a single chat interface.

Getting started

  1. The app opens to the Chat view with a simulated WordPress site and floating chat widget.
  2. Try the chat widget — select context (agency, page, section) and talk to the agent.
  3. Navigate to Tickets to see development requests and bug reports.
  4. Review code changes in the Code Changes view with diff display.
  5. Follow the release process in the Pipeline view.
  6. Check accessibility on the Audit tab.

Six views

View 1

Chat

Conversation with the AI agent in the context of a simulated WordPress site.

  • Context selector — choose agency, page and section you're referring to.
  • Floating chat widget with typing animation and reactions.
  • Background shows a simulated site illustrating the change targets.
View 2

Tickets

Development requests and bug reports with status filters.

  • Filter tickets: new, in progress, review, approved, deployed.
  • Ticket statistics and summary dashboard.
  • Conversation modal for individual ticket handling.
View 3

Code Changes

PR and merge request review view.

  • Diff view of code changes line by line.
  • Review comments and approval process.
View 4

Pipeline

CI/CD pipeline and release process tracking.

  • Kanban-style pipeline view by stage.
  • Release status and progress visually displayed.
View 5

Audit

Accessibility and usability audit results.

  • Audit result import and visualization.
  • WCAG compliance check per page.
View 6

Analytics

Development metrics and drill-down charts.

  • Ticket processing times, volume and trends.
  • Drill-down charts for deeper analysis.
Tip: Start with the Chat view — select a context and ask the agent. The agent can advise on ticket status, code changes and accessibility issues.

Use cases

Developer

"What tickets are assigned to me?"

Open Tickets → filter "in progress" → see your active development requests and bugs.

Project coordinator

"What's the release status?"

Open Pipeline → see where each change is on its way to production.

Accessibility specialist

"Are there accessibility issues on the site?"

Open Audit → check WCAG results per page and prioritize fixes.

Good to know

  • The app simulates the Justice Ministry WordPress multisite environment.
  • The chat agent, tickets and audit results are generated for demo purposes.
  • Chat also supports voice input (WebkitSpeechRecognition).
  • The app runs entirely in the browser with no server required.
