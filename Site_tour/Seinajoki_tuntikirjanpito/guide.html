Miksi tämä sovellus on tehty?

Tuntien kirjaaminen on pakollista mutta tylsää — eikä perinteinen Excel-taulukko kerro missä aika oikeasti meni. Tuntikirjanpito yhdistää viikkokohtaisen tuntikirjauksen Git-historiaan, Jira-tehtäviin ja kalenteriin, jotta näet yhdellä silmäyksellä mitä teit ja voit kirjata tunnit luotettavasti sen perusteella.

Näin pääset alkuun

  1. Sovellus avautuu viikkonäkymään jossa näet kuluvan viikon työtuntitaulukon.
  2. Ensimmäisellä kerralla avautuu tervetuloikkuna — valitse "Arvaa tunnit" niin sovellus täyttää viikon automaattisesti, tai aloita tyhjältä pohjalta.
  3. Syötä tunnit klikkaamalla soluja ja kirjoittamalla tuntimäärä (esim. 7.5).
  4. Vaihda välilehtien avulla tunnit-, git-, jira- ja kalenterinäkymien välillä.
  5. Siirry Analytiikka-näkymään nähdäksesi kokonaiskuva projekteista ja ajankäytöstä.

Kaksi päänäkymää

Näkymä 1

Viikko (tuntitaulukko)

Päätyökalu tuntien kirjaamiseen. Näyttää viikon ma–su sarakkeina ja projektit riveinä.

  • Klikkaa solua → syötä tunnit. Yhteenlaskut päivittyvät automaattisesti.
  • Summasarake oikealla näyttää kunkin projektin viikkosumman.
  • Alarivi näyttää päivän kokonaistunnit — ylitykset korostetaan punaisella.
  • Navigoi viikkojen välillä nuolipainikkeilla tai palaa tähän viikkoon "Tänään"-painikkeella.
  • "Kopioi ed. viikko" kopioi edellisen viikon tunnit pohjaksi.
Alavälilehdet

Git, Jira ja Kalenteri

Tuntitaulukon yläpuolella olevat välilehdet näyttävät viikon aktiviteetin eri lähteistä:

  • Git — viikon commitit projekteittain ja päivittäin. Näet mitä koodia kirjoitit minäkin päivänä.
  • Jira — viikon tehtävät ja niiden tilat (To Do, In Progress, Code Review, Done).
  • Kalenteri — viikon palaverit ja tapahtumat aikoineen.

Nämä auttavat muistamaan mitä teit ja kirjaamaan tunnit oikein.

Näkymä 2

Analytiikka

Kokonaiskuva ajankäytöstä kaavioiden ja porautumisen avulla.

  • Aikarajaus: vuosi, neljännes, kuukausi tai viikko. Navigoi jaksojen välillä nuolilla.
  • Graafin tarkkuus: päivät, viikot tai kuukaudet.
  • Donitsikaavio näyttää tuntien jakautumisen projekteittain.
  • Palkkikaavio näyttää tuntien kehityksen ajanjaksoittain.
  • Klikkaa kaaviota → porautumisnäkymä avautuu yksityiskohtaisella listauksella.
Vinkki: Vaihda ensin Git- tai Kalenteri-välilehdelle muistaaksesi mitä teit viikolla, ja siirry sitten takaisin Tunnit-välilehdelle kirjaamaan tunnit. Näin kirjaukset vastaavat todellisuutta.

Käyttötapaukset

Konsultti

"Paljonko tein projektia X tässä kuussa?"

Siirry Analytiikka-näkymään → valitse aikarajaukseksi "Kuukausi" → katso donitsikaavio tai klikkaa projektia porautuaksesi päiväkohtaisiin tunteihin.

Perjantai-kirjaaja

"En muista mitä tein maanantaina"

Vaihda Git-välilehdelle → näet päivän commitit. Tarkista myös Kalenteri-välilehti palavereiden osalta. Palaa Tunnit-välilehdelle ja kirjaa tuntimäärät.

Projektipäällikkö

"Kuinka paljon resursseja meni tähän sprinttiin?"

Avaa Analytiikka → valitse aikarajaukseksi "Viikko" ja navigoi sprintin viikoille → vertaile projektijakaumaa viikkojen välillä.

Uusi työntekijä

"Miten kirjaan tuntini?"

Sovellus avautuu viikkonäkymään → valitse tervetuloikkunasta "Arvaa tunnit" tai "Aloita tyhjänä" → klikkaa soluja ja syötä tuntimäärät → paina "Tallenna muutokset".

Laskuttaja

"Paljonko asiakkaalle A tehtiin Q1:llä?"

Avaa Analytiikka → aikarajaus "Neljännes" → navigoi Q1:lle → klikkaa asiakkaan projektia donitsikaaviossa → porautumisnäkymä näyttää viikkokohtaisen erittelyn.

Hyödyllistä tietää

  • Sovellus sisältää demoaineiston useilla projekteilla, git-commiteilla, jira-tehtävillä ja kalenterimerkinnöillä.
  • "Arvaa tunnit" -toiminto generoi realistiset tuntikirjaukset automaattisesti git- ja kalenteridatan perusteella.
  • Tiedot tallentuvat selaimen muistiin (localStorage) — ne säilyvät sivun uudelleenlatauksessa mutta eivät eri selainten välillä.
  • Viikonloppupäivät näkyvät harmaalla taustalla ja kuluvaa päivää korostetaan sinisellä.
  • Jos päivän tunnit ylittävät normaalin työajan, summa näkyy punaisella.
  • Sovellus toimii kokonaan selaimessa ilman palvelinta.

Why was this app built?

Logging hours is mandatory but tedious — and a traditional Excel spreadsheet doesn't tell you where the time actually went. Time Tracking combines weekly hour logging with Git history, Jira tasks, and your calendar, so you can see at a glance what you did and log your hours reliably based on that.

Getting started

  1. The app opens to the week view showing the current week's timesheet.
  2. On first visit, a welcome dialog appears — choose "Guess hours" to have the app fill in the week automatically, or start with a blank sheet.
  3. Enter hours by clicking cells and typing the amount (e.g. 7.5).
  4. Switch between hours, Git, Jira, and calendar views using the tabs.
  5. Go to the Analytics view to see the big picture of projects and time usage.

Two main views

View 1

Week (timesheet)

The main tool for logging hours. Shows the week Mon–Sun as columns and projects as rows.

  • Click a cell → enter hours. Totals update automatically.
  • Sum column on the right shows each project's weekly total.
  • Bottom row shows daily total hours — overages are highlighted in red.
  • Navigate between weeks with arrow buttons or return to this week with the "Today" button.
  • "Copy prev. week" copies the previous week's hours as a starting point.
Sub-tabs

Git, Jira, and Calendar

The tabs above the timesheet show the week's activity from different sources:

  • Git — the week's commits by project and day. See what code you wrote each day.
  • Jira — the week's tasks and their statuses (To Do, In Progress, Code Review, Done).
  • Calendar — the week's meetings and events with times.

These help you remember what you did and log hours accurately.

View 2

Analytics

A big-picture view of time usage through charts and drill-downs.

  • Time range: year, quarter, month, or week. Navigate between periods with arrows.
  • Chart granularity: days, weeks, or months.
  • Donut chart shows hour distribution by project.
  • Bar chart shows hour trends over time periods.
  • Click a chart → a drill-down view opens with a detailed breakdown.
Tip: First switch to the Git or Calendar tab to recall what you did during the week, then go back to the Hours tab to log your time. This way your entries match reality.

Use cases

Consultant

"How much did I work on project X this month?"

Go to the Analytics view → set the time range to "Month" → check the donut chart or click a project to drill down to daily hours.

Friday logger

"I don't remember what I did on Monday"

Switch to the Git tab → you'll see the day's commits. Also check the Calendar tab for meetings. Return to the Hours tab and log your time.

Project manager

"How many resources went into this sprint?"

Open Analytics → set the time range to "Week" and navigate to the sprint weeks → compare project distribution across weeks.

New employee

"How do I log my hours?"

The app opens to the week view → choose "Guess hours" or "Start empty" from the welcome dialog → click cells and enter hours → press "Save changes".

Billing

"How much was done for client A in Q1?"

Open Analytics → time range "Quarter" → navigate to Q1 → click the client's project in the donut chart → the drill-down view shows a week-by-week breakdown.

Good to know

  • The app includes demo data with multiple projects, Git commits, Jira tasks, and calendar entries.
  • The "Guess hours" feature automatically generates realistic time entries based on Git and calendar data.
  • Data is saved in browser storage (localStorage) — it persists across page reloads but not across different browsers.
  • Weekend days have a grey background and the current day is highlighted in blue.
  • If daily hours exceed normal working time, the total appears in red.
  • The app runs entirely in the browser with no server required.
