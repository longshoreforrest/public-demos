Miksi tämä sovellus on tehty?

Site Tour -kokonaisuus sisältää kymmeniä erillisiä web-sovelluksia, mutta niiden tietoturvaa ei ole systemaattisesti arvioitu. Tämä sovellus käy läpi kaikkien sovellusten lähdekoodin ja raportoi löydökset OWASP Top 10 -viitekehyksen mukaisesti — niin tietoturvatilanne selviää yhdellä silmäyksellä ilman manuaalista koodiauditointia.

Näin pääset alkuun

  1. Sovellus avautuu Kojelauta-näkymään, jossa näet yhteenvedon tietoturvalöydöksistä interaktiivisina kaavioina.
  2. Vaihda näkymää yläosan välilehdillä: Kojelauta, Havainnot tai Sovellukset.
  3. Käytä suodatinpainikkeita rajataksesi näkymää toimipisteen, sovelluksen, OWASP-kategorian tai vakavuuden mukaan.
  4. Klikkaa mitä tahansa kaaviota porautuaksesi syvemmälle — filtterit päivittyvät automaattisesti.

Näkymät

Välilehti 1

Kojelauta

Interaktiivinen analytiikkanäkymä neljällä kaaviolla:

  • OWASP-kategorialla — donitsikaavio, joka näyttää löydösten jakauman OWASP Top 10 -kategorioittain.
  • Vakavuudella — donitsikaavio vakavuustasojen mukaan (Kriittinen, Korkea, Keskitaso, Matala, Info).
  • Sovelluksella — vaakapalkki, jossa jokainen sovellus näkyy löydösmäärän mukaan.
  • Vakavuus sovelluksittain — pinottu palkki, joka näyttää vakavuusjakauman sovelluksittain.

Porautuminen: Klikkaa mitä tahansa kaavion segmenttiä → vastaava suodatin asettuu ja kaikki kaaviot päivittyvät. Klikkaa samaa segmenttiä uudelleen poistaaksesi suodattimen.

Välilehti 2

Havainnot

Yksityiskohtainen taulukkonäkymä kaikista löydöksistä:

  • Sarakkeet: vakavuus, OWASP-kategoria, sovellus, tiedosto ja otsikko.
  • Lajittele klikkaamalla sarakeotsikkoa.
  • Hae vapaatekstihaulla löydösten sisällöstä.
  • Klikkaa riviä → avautuu yksityiskohtainen sivupaneeli, jossa kuvaus, suositus ja linkki OWASP-lähteeseen.
Välilehti 3

Sovellukset

Korttinäkymä kaikista skannatuista sovelluksista:

  • Jokainen kortti näyttää sovelluksen nimen, löydösmäärän ja riskipisteytyksen.
  • Värikoodatut vakavuusbadget kertovat löydösten jakauman.
  • Klikkaa korttia → siirryt Havainnot-näkymään suodatettuna kyseiseen sovellukseen.

Excel-vienti

Klikkaa otsikkorivin Excel-painiketta ladataksesi suodatetut löydökset Excel-muodossa. Tiedosto sisältää kaksi välilehteä:

Käyttötapaukset

Tietoturvavastaava

"Mikä on sovellusten tietoturvatilanne?"

Avaa kojelauta → tarkastele riskipisteytystä ja vakavuusjakaumaa. Poraudu kriittisiin ja korkeisiin löydöksiin. Vie Excel-raportti johdolle.

Kehittäjä

"Mitä minun sovelluksessani pitää korjata?"

Valitse sovelluksesi Sovellus-suodattimesta → näet kaikki kyseisen sovelluksen löydökset. Klikkaa löydöstä → lue kuvaus, suositus ja OWASP-lähde.

Projektipäällikkö

"Minkä tiimin sovelluksissa on eniten riskejä?"

Käytä Toimipiste-suodatinta rajataksesi kaupungin mukaan → vertaile sovelluksia Sovellukset-näkymässä riskipisteytyksen perusteella.

Auditoija

"Tarvitsen kattavan raportin löydöksistä"

Vie kaikki löydökset Exceliin → saat värikoodatun taulukon ja yhteenvedon. Suodata tarvittaessa tiettyihin kategorioihin ennen vientiä.

Hyödyllistä tietää

  • Riskipisteytys lasketaan painottamalla vakavuustasoja: Kriittinen = 10, Korkea = 5, Keskitaso = 2, Matala = 0.5, Info = 0.
  • Kielivalinta (FI/EN) tallentuu selaimen muistiin ja pysyy samana kaikissa Site Tour -sovelluksissa.
  • Kaavioiden porautuminen toimii toggle-periaatteella: ensimmäinen klikkaus suodattaa, toinen klikkaus poistaa suodattimen.
  • Löydösten yksityiskohtanäkymässä on suora linkki alkuperäiseen OWASP-lähteeseen.
  • Sovellus toimii kokonaan selaimessa — tietoja ei lähetetä mihinkään palvelimelle.

Why does this app exist?

The Site Tour collection includes dozens of separate web applications, but their security has not been systematically assessed. This application scans the source code of all applications and reports findings using the OWASP Top 10 framework — giving you a complete security overview at a glance without manual code auditing.

Getting started

  1. The app opens in the Dashboard view, showing a summary of security findings as interactive charts.
  2. Switch views using the tabs at the top: Dashboard, Findings or Applications.
  3. Use the filter buttons to narrow the view by location, application, OWASP category or severity.
  4. Click any chart segment to drill down — filters update automatically.

Views

Tab 1

Dashboard

An interactive analytics view with four charts:

  • By OWASP Category — doughnut chart showing findings distributed across OWASP Top 10 categories.
  • By Severity — doughnut chart by severity level (Critical, High, Medium, Low, Info).
  • By Application — horizontal bar chart showing finding counts per application.
  • Severity per Application — stacked bar chart showing severity breakdown per app.

Drill-down: Click any chart segment → the corresponding filter is set and all charts update. Click the same segment again to remove the filter.

Tab 2

Findings

A detailed table view of all findings:

  • Columns: severity, OWASP category, application, file and title.
  • Sort by clicking column headers.
  • Search with free-text across finding content.
  • Click a row → opens a detail panel with description, recommendation and a link to the OWASP source.
Tab 3

Applications

A card view of all scanned applications:

  • Each card shows the application name, finding count and risk score.
  • Colour-coded severity badges show the distribution of findings.
  • Click a card → navigates to the Findings view filtered to that application.

Filters

The application has four filter rows that can be hidden and shown with the toggle button at the top:

FilterFunction
LocationFilter findings by city (Helsinki, Joensuu, Kuopio, etc.)
ApplicationSelect a specific application to examine
OWASPFilter by a specific OWASP Top 10 category
SeverityFilter by severity level

Each filter row includes an "All" button that removes that filter. The "Reset all" button clears all active filters at once.

Tip: You can also filter by clicking the severity cards at the top of the dashboard. When filters are collapsed, the active filter count is shown as a badge on the toggle button.

Click the Excel button in the header to download the filtered findings in Excel format. The file contains two sheets:

  • Security Findings — all findings colour-coded by severity.
  • Summary — summary by severity and OWASP category.

Use cases

Security officer

"What is the security posture of our applications?"

Open the dashboard → review the risk score and severity distribution. Drill into critical and high findings. Export an Excel report for management.

Developer

"What needs fixing in my application?"

Select your application from the Application filter → see all findings for that app. Click a finding → read the description, recommendation and OWASP source.

Project manager

"Which team's applications have the most risk?"

Use the Location filter to narrow by city → compare applications in the Applications view by risk score.

Auditor

"I need a comprehensive report of findings"

Export all findings to Excel → you get a colour-coded table and summary. Filter to specific categories before exporting if needed.

Good to know

  • Risk score is calculated by weighting severity levels: Critical = 10, High = 5, Medium = 2, Low = 0.5, Info = 0.
  • Language selection (FI/EN) is saved in the browser and persists across all Site Tour applications.
  • Chart drill-down works as a toggle: first click filters, second click removes the filter.
  • The finding detail panel includes a direct link to the original OWASP source.
  • The application runs entirely in the browser — no data is sent to any server.
