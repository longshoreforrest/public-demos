WordPress-sivustojen QA-havaintokeskus

Miksi tämä sovellus on tehty?

WordPress-multisite-ympäristön testiautomaatio tuottaa satoja havaintoja viikossa — eri kategorioissa, vakavuusasteissa ja virastoissa. Manuaalinen priorisointi ja juurisyyanalyysi on hidasta. Havaintokeskus kokoaa testiautomaation tulokset, tunnistaa kuviot automaattisesti ja tarjoaa AI-agentin suositukset korjauksiin.

Näin pääset alkuun

  1. Sovellus avautuu Dashboard-näkymään, jossa näet havaintojen kokonaiskuvan tilastoina ja kaavioina.
  2. Klikkaa tilastokortteja porautuaksesi yksittäisiin kategorioihin tai vakavuustasoihin.
  3. Siirry Havainnot-välilehteen selataksesi yksittäisiä havaintoja ja suodattaaksesi niitä.
  4. Tarkastele AI-agentin analyysejä Agenttianalyysi-näkymässä.
  5. Seuraa trendejä ja korrelaatioita Trendit-välilehdellä.

Neljä näkymää

Näkymä 1

Dashboard

Yleiskatsaus havaintoihin tilastokortteina ja kaavioina.

  • Tilastokortit — havaintojen lukumäärä kategorioittain ja vakavuuksittain. Klikkaa korttia porautuaksesi.
  • Aikajanakaavio näyttää havaintojen kehityksen yli ajan.
  • Jakaumakaaviot kategorioista, vakavuuksista ja virastoista.
Näkymä 2

Havainnot

Yksittäisten havaintojen selaus ja suodatus.

  • Monipuoliset suodattimet — kategoria, vakavuus, tila, virasto, työkalu ja komponentti.
  • Hakutoiminto havaintojen kuvauksista ja otsikoista.
  • Klikkaa havaintoa nähdäksesi tarkemmat tiedot ja agenttianalyysin.
  • Sivutus suurille tietomäärille.
Näkymä 3

Agenttianalyysi

AI-agentin tuottamat juurisyyanalyysit ja suositukset.

  • Tunnistetut kuviot — Salesforce-integraatioketjut, saavutettavuusvelka, suorituskykyongelmat.
  • Juurisyyanalyysi jokaiselle havainnolle.
  • Korjaussuositukset priorisoituna.
Näkymä 4

Trendit ja korrelaatiot

Viikko- ja kuukausitrendit sekä vertailut.

  • Trendikaaviot havainnoista viikko- ja kuukausitasolla.
  • Ratkaisuaikojen analytiikka — kuinka nopeasti havainnot ratkaistaan.
  • Sisäisten ja ulkoisten ongelmien vertailu (Salesforce, DVV, Suomi.fi).
Vinkki: Aloita Dashboardista kokonaiskuvan saamiseksi. Jos näet kriittisiä havaintoja, klikkaa korttia — pääset suoraan suodatettuun listaan. Agenttianalyysi kertoo juurisyyn ja ehdottaa korjausta.

Käyttötapaukset

QA-insinööri

"Mitä uusia havaintoja tänään on tullut?"

Avaa Dashboard → tilastokortit näyttävät tuoreet havainnot. Klikkaa korttia tai siirry Havainnot-välilehdelle suodattamaan tila: "avoin".

Kehitystiimin vetäjä

"Mitkä ongelmat toistuvat ja mistä ne johtuvat?"

Avaa Agenttianalyysi → tunnistetut kuviot kertovat toistuvat ongelmat ja niiden juurisyyt. Tarkista Trendit-välilehdeltä onko suunta paranemaan.

Projektipäällikkö

"Johtuvatko ongelmat meistä vai ulkoisista järjestelmistä?"

Avaa Trendit → sisäisten ja ulkoisten ongelmien vertailu näyttää jakautumisen. Ulkoiset riippuvuudet (Salesforce, DVV, Suomi.fi) erottuvat selvästi.

Hyödyllistä tietää

  • Havaintokeskus kattaa oikeusministeriön hallinnonalan WordPress-sivustot.
  • Havaintodata on demoa varten generoitua — oikeassa käytössä se tulisi testiautomaatiojärjestelmistä.
  • AI-agentin analyysit tunnistavat kuvioita kuten WCAG-saavutettavuusvelka, OWASP-turvallisuusongelmat ja integraatioketjujen häiriöt.
  • Kaaviot ovat interaktiivisia — klikkaa datapistettä porautuaksesi tarkempiin tietoihin.
  • Sovellus toimii kokonaan selaimessa ilman palvelinta.

Why was this app built?

A WordPress multisite test automation environment produces hundreds of observations per week — across different categories, severity levels and agencies. Manual prioritization and root cause analysis is slow. The observatory gathers test automation results, automatically identifies patterns and provides AI agent recommendations for fixes.

Getting started

  1. The app opens to the Dashboard showing the overall observation picture as statistics and charts.
  2. Click stat cards to drill down into individual categories or severity levels.
  3. Navigate to Observations to browse and filter individual findings.
  4. Review AI agent analyses in the Agent Analysis view.
  5. Follow trends and correlations on the Trends tab.

Four views

View 1

Dashboard

Overview of observations as stat cards and charts.

  • Stat cards — observation counts by category and severity. Click a card to drill down.
  • Timeline chart shows observation trends over time.
  • Distribution charts for categories, severities and agencies.
View 2

Observations

Browse and filter individual observations.

  • Multiple filters — category, severity, status, agency, tool and component.
  • Search observation descriptions and titles.
  • Click an observation to see details and agent analysis.
  • Pagination for large datasets.
View 3

Agent Analysis

AI agent-generated root cause analyses and recommendations.

  • Detected patterns — Salesforce integration chains, accessibility debt, performance issues.
  • Root cause analysis for each observation.
  • Fix recommendations prioritized.
View 4

Trends & Correlations

Weekly and monthly trends with comparisons.

  • Trend charts at weekly and monthly levels.
  • Resolution time analytics — how quickly observations are resolved.
  • Internal vs. external issue comparison (Salesforce, DVV, Suomi.fi).
Tip: Start with the Dashboard for the big picture. If you see critical observations, click the card — you'll go straight to a filtered list. Agent Analysis tells you the root cause and suggests a fix.

Use cases

QA Engineer

"What new observations came in today?"

Open Dashboard → stat cards show fresh observations. Click a card or go to Observations and filter by status: "open".

Development team lead

"Which issues keep recurring and why?"

Open Agent Analysis → detected patterns reveal recurring issues and their root causes. Check Trends to see if things are improving.

Project manager

"Are the issues caused by us or external systems?"

Open Trends → internal vs. external comparison shows the split. External dependencies (Salesforce, DVV, Suomi.fi) stand out clearly.

Good to know

  • The observatory covers Justice Ministry WordPress sites.
  • Observation data is generated for demo purposes — in real use it would come from test automation systems.
  • AI agent analyses detect patterns like WCAG accessibility debt, OWASP security issues and integration chain disruptions.
  • Charts are interactive — click a data point to drill into details.
  • The app runs entirely in the browser with no server required.
