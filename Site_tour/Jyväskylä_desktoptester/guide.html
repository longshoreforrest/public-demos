Käyttöohje

Tämä sovellus esittelee miten desktop-sovelluksia (Outlook, SAP, Excel) voidaan testata automaattisesti kuvantunnistuksen ja AI:n avulla. Mukana on interaktiivinen web-demo ja oikeat Python-skriptit.

Web-sovelluksen välilehdet

Simulaattori

Interaktiivinen animaatio joka näyttää miten PyAutoGUI-pohjainen botti automatisoisi Outlook-sovelluksen käyttöä. Paina "Aloita testi" ja seuraa miten botti:

  • Käynnistää sovelluksen
  • Etsii UI-elementtejä kuvantunnistuksella
  • Syöttää tietoja ja klikkaa nappeja
  • Ottaa kuvakaappauksia todisteiksi

Sivun alaosassa on myös ohjeet oikeiden Python-skriptien ajamiseen omalla koneella.

Yleiskatsaus

Selittää ongelman (miksi desktop-testaus on vaikeaa) ja esittelee kaksi päälähestymistapaa: kuvantunnistuspohjainen ja saavutettavuuspuupohjainen.

Lähestymistavat

Vertailutaulukko kuudesta eri teknologiasta (PyAutoGUI, Claude Computer Use, SikuliX, WinAppDriver, Appium, Power Automate). Jokaiselle on listattu vahvuudet ja heikkoudet.

Esimerkki

Kolme konkreettista Python-koodiesimerkikä samasta testitapauksesta ("avaa Outlook, lue sähköposti") toteutettuna kolmella eri teknologialla.

Testaajan työnkulku

Askel askeleelta -ohje: asennuksesta, referenssikuvien ottamisesta, skriptin kirjoittamisesta ja CI/CD-integraatiosta.

Rajoitteet

Rehellinen kuvaus desktop-testausautomaation haasteista: resoluutioriippuvuus, ei headless-tukea, ajoitusongelmat, ylläpitokustannukset ja tietoturva.

Oikeiden skriptien ajaminen

Vaihtoehto 1: PyAutoGUI-demo (ei API-avainta)

Yksinkertaisin tapa kokeilla. Avaa Laskin-sovelluksen ja laskee 42 + 58.

pip install pyautogui pillow python scripts/demo_pyautogui.py

Vaihtoehto 2: Claude Computer Use -demo

AI-pohjainen demo. Vaatii Anthropic API-avaimen.

pip install anthropic pyautogui pillow export ANTHROPIC_API_KEY="sk-ant-..." python scripts/demo_claude_computeruse.py

Voit myös antaa oman tehtävän:

python scripts/demo_claude_computeruse.py \ --task "Avaa TextEdit ja kirjoita Hello World"
Varoitus: Skriptit ottavat hiiren ja näppäimistön haltuunsa ajon ajaksi. Turvamekanismi: siirrä hiiri ruudun nurkkaan, niin skripti pysähtyy välittömästi (PyAutoGUI FAILSAFE).

Esivaatimukset

  • Python 3.8+ asennettu koneelle
  • macOS: Anna terminaalille Screen Recording -oikeus (System Settings → Privacy & Security → Screen Recording)
  • Windows: Ei erityisvaatimuksia
  • Claude-demolle: Anthropic API-avain (sk-ant-...)

Kielenvaihtaja

Pääsovelluksessa oikean yläkulman FI/EN-painikkeella voi vaihtaa kielen suomen ja englannin välillä.

Vinkki: Claude Computer Use -lähestymistapa on joustavin: se ei tarvitse referenssikuvia eikä kovakoodattuja koordinaatteja. AI sopeutuu automaattisesti kun sovelluksen UI muuttuu.

User Guide

This application demonstrates how desktop applications (Outlook, SAP, Excel) can be tested automatically using image recognition and AI. Includes an interactive web demo and real Python scripts.

Web Application Tabs

Simulator

An interactive animation showing how a PyAutoGUI-based bot would automate Outlook. Press "Start Test" and watch the bot:

  • Launch the application
  • Find UI elements using image recognition
  • Enter data and click buttons
  • Take screenshots as evidence

The bottom of the page also contains instructions for running the real Python scripts on your own machine.

Overview

Explains the problem (why desktop testing is hard) and presents two main approaches: image recognition-based and accessibility tree-based.

Approaches

Comparison table of six different technologies (PyAutoGUI, Claude Computer Use, SikuliX, WinAppDriver, Appium, Power Automate). Each includes strengths and weaknesses.

Example

Three concrete Python code examples of the same test case ("open Outlook, read email") implemented with three different technologies.

Tester Workflow

Step-by-step guide: from installation, capturing reference images, writing scripts, to CI/CD integration.

Limitations

An honest description of desktop test automation challenges: resolution dependency, no headless support, timing issues, maintenance costs, and security.

Running the Real Scripts

Option 1: PyAutoGUI Demo (no API key needed)

The simplest way to try it out. Opens Calculator and computes 42 + 58.

pip install pyautogui pillow python scripts/demo_pyautogui.py

Option 2: Claude Computer Use Demo

AI-based demo. Requires an Anthropic API key.

pip install anthropic pyautogui pillow export ANTHROPIC_API_KEY="sk-ant-..." python scripts/demo_claude_computeruse.py

You can also provide a custom task:

python scripts/demo_claude_computeruse.py \ --task "Open TextEdit and type Hello World"
Warning: The scripts take control of your mouse and keyboard during execution. Safety mechanism: move the mouse to a screen corner and the script stops immediately (PyAutoGUI FAILSAFE).

Prerequisites

  • Python 3.8+ installed on your machine
  • macOS: Grant Screen Recording permission to your terminal (System Settings → Privacy & Security → Screen Recording)
  • Windows: No special requirements
  • For Claude demo: Anthropic API key (sk-ant-...)

Language Switcher

In the main application, use the FI/EN button in the top-right corner to switch between Finnish and English.

Tip: The Claude Computer Use approach is the most flexible: it doesn't need reference images or hardcoded coordinates. The AI automatically adapts when the application UI changes.
