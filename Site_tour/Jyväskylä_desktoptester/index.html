Miten testata desktop-sovelluksia automaattisesti?

Desktop-sovellusten (Outlook, SAP, Excel, Teams) automaatiotestaus on perinteisesti ollut vaikeaa, koska sovelluksilla ei ole standardoitua API:a testaukseen kuten web-sovelluksilla. Tämä sivu esittelee modernit ratkaisut tähän ongelmaan.

Ongelma

Web-sovelluksia voi testata Seleniumilla tai Playwrightilla, koska ne manipuloivat DOM-puuta. Desktop-sovelluksilla ei ole vastaavaa universaalia rajapintaa. Miten siis automatisoida esimerkiksi: "Avaa Outlook, kirjaudu sisään, avaa ensimmäinen sähköposti, klikkaa Vastaa"?

Kaksi päälähestymistapaa

🖼️ Kuvantunnistus (Visual)

Automaatio "näkee" ruudun kuvakaappauksena ja etsii nappeja, kenttiä ja ikkunoita kuvatunnistuksella (template matching tai AI). Ei tarvitse tietää sovelluksen sisäistä rakennetta.

Universaali Ei vaadi API:a Herkempi muutoksille

🌳 Saavutettavuuspuu (Accessibility)

Käytetään käyttöjärjestelmän saavutettavuusrajapintaa (UI Automation, AT-SPI) elementtien löytämiseen. Toimii kuin Selenium, mutta desktop-sovelluksille.

🐍 PyAutoGUI + OpenCV

Python-kirjasto joka hallitsee hiirtä ja näppäimistöä sekä ottaa kuvakaappauksia. OpenCV lisää kuvatunnistuksen: voit etsiä ruudulta tiettyä nappia tai ikonia kuvan perusteella.

Vahvuudet

  • Helppo asentaa: pip install pyautogui
  • Cross-platform (Win/Mac/Linux)
  • Toimii kaikkien sovellusten kanssa
  • Hyvä dokumentaatio ja yhteisö

Heikkoudet

  • Herkkä resoluutio- ja teemamuutoksille
  • Ei ymmärrä mitä näkee (ei AI:ta)
  • Tarvitsee referenssikuvia ylläpidettäväksi
  • Yksi näyttö kerrallaan

🧠 Claude Computer Use

Anthropicin Claude-mallin computer use -ominaisuus: AI näkee ruudun kuvakaappauksena ja osaa päätellä mitä klikata, minne kirjoittaa ja miten navigoida. Ei tarvitse referenssikuvia - AI ymmärtää UI:n semanttisesti.

Vahvuudet

  • Ymmärtää UI:n merkityksen, ei vain pikseleitä
  • Ei tarvitse referenssikuvia
  • Sopeutuu UI-muutoksiin automaattisesti
  • Voi suorittaa monimutkaisia tehtäviä luonnollisella kielellä

Heikkoudet

  • API-kustannukset (kuvakaappaus per pyyntö)
  • Hitaampi kuin perinteinen automaatio
  • Vaatii internet-yhteyden (API)
  • Tulos ei ole 100% deterministinen

🪟 WinAppDriver / Appium

Microsoftin WinAppDriver käyttää Windows UI Automation -rajapintaa: sovelluksen elementit löydetään saavutettavuuspuusta nimen, AutomationId:n tai ControlType:n perusteella. Toimii kuin Selenium, mutta natiivisovelluksille.

Vahvuudet

  • Vakaa - ei riipu pikselikoordinaateista
  • Nopea suoritus
  • Integroituu Selenium-ekosysteemiin
  • Tarkka elementtien tunnistus

Heikkoudet

  • Vain Windows
  • Ei toimi kaikilla sovelluksilla (esim. legacy)
  • Tarvitsee Inspect-työkalun elementtien löytämiseen
  • Jotkut sovellukset piilottavat elementit

Konkreettinen esimerkki: Outlook-testaus

Alla on kolme eri tapaa toteuttaa sama testitapaus. Jokainen lähestymistapa ratkaisee saman ongelman eri tavalla.

Testitapaus: "Avaa Outlook, kirjaudu sisään, lue ensimmäinen sähköposti"

1. PyAutoGUI + OpenCV

Perinteinen kuvantunnistuspohjainen lähestymistapa. Tarvitsee referenssikuvat UI-elementeistä.

# outlook_test.py - Desktop-testaus PyAutoGUI:lla import pyautogui import time import subprocess # Aseta turvamekanismi: siirrä hiiri nurkkaan pysäyttääksesi pyautogui.FAILSAFE = True pyautogui.PAUSE = 0.5 # 0.5s viive jokaisen toiminnon väliin def wait_for_image(image_path, timeout=30, confidence=0.8): """Odota kunnes kuva löytyy ruudulta.""" start = time.time() while time.time() - start < timeout: location = pyautogui.locateOnScreen( image_path, confidence=confidence ) if location: return pyautogui.center(location) time.sleep(1) raise TimeoutError(f"Kuvaa {image_path} ei löytynyt {timeout}s sisällä") def test_outlook_read_email(): # 1. Käynnistä Outlook subprocess.Popen("outlook.exe") print("[INFO] Käynnistetään Outlook...") # 2. Odota että Outlook avautuu (etsi tunnettua ikonia) inbox_btn = wait_for_image("images/outlook_inbox_icon.png") print("[OK] Outlook avautui") # 3. Klikkaa Saapuneet-kansiota pyautogui.click(inbox_btn) time.sleep(2) # 4. Jos kirjautumisikkuna ilmestyy try: login_field = wait_for_image( "images/email_field.png", timeout=5 ) pyautogui.click(login_field) pyautogui.typewrite("testaaja@firma.fi", interval=0.05) pyautogui.press("enter") print("[OK] Kirjautumistiedot syötetty") except TimeoutError: print("[INFO] Kirjautumista ei tarvittu") # 5. Klikkaa ensimmäistä sähköpostia first_email = wait_for_image("images/unread_email_icon.png") pyautogui.click(first_email) time.sleep(1) print("[OK] Ensimmäinen sähköposti avattu") # 6. Ota kuvakaappaus todisteeksi screenshot = pyautogui.screenshot() screenshot.save("test_results/outlook_email_opened.png") print("[OK] Kuvakaappaus tallennettu") if __name__ == "__main__": test_outlook_read_email()

2. Claude Computer Use

AI-pohjainen lähestymistapa: kerro luonnollisella kielellä mitä haluat tehdä, ja Claude suorittaa sen.

# outlook_test_claude.py - Desktop-testaus Claude Computer Use:lla import anthropic import base64 import subprocess from PIL import ImageGrab client = anthropic.Anthropic() def take_screenshot(): """Ota kuvakaappaus ja palauta base64-muodossa.""" img = ImageGrab.grab() import io buffer = io.BytesIO() img.save(buffer, format="PNG") return base64.b64encode(buffer.getvalue()).decode() def execute_action(action): """Suorita Clauden ehdottama toiminto.""" if action["type"] == "click": import pyautogui pyautogui.click(action["x"], action["y"]) elif action["type"] == "type": import pyautogui pyautogui.typewrite(action["text"]) elif action["type"] == "key": import pyautogui pyautogui.press(action["key"]) def test_outlook_with_claude(): # 1. Käynnistä Outlook subprocess.Popen("outlook.exe") # 2. Anna tehtävä Claudelle luonnollisella kielellä task = """ Olet testausautomaatiobotti. Suorita seuraavat askeleet: 1. Odota että Outlook on avautunut 2. Jos kirjautumisikkuna näkyy, kirjaudu sisään 3. Klikkaa Saapuneet-kansiota (Inbox) 4. Avaa ensimmäinen lukematon sähköposti 5. Raportoi sähköpostin otsikko ja lähettäjä """ messages = [{"role": "user", "content": [ {"type": "text", "text": task}, {"type": "image", "source": { "type": "base64", "media_type": "image/png", "data": take_screenshot() }} ]}] # 3. Claude analysoi ruudun ja ehdottaa toimintoja response = client.messages.create( model="claude-sonnet-4-20250514", max_tokens=1024, tools=[{ "type": "computer_20250124", "name": "computer", "display_width_px": 1920, "display_height_px": 1080, "display_number": 1 }], messages=messages ) print("Claude vastasi:", response.content) if __name__ == "__main__": test_outlook_with_claude()

3. WinAppDriver (saavutettavuuspuu)

Selenium-tyylinen lähestymistapa: elementit löydetään nimellä ja AutomationId:lla.

# outlook_test_winappdriver.py - WinAppDriver from appium import webdriver from appium.options.windows import WindowsOptions def test_outlook_winappdriver(): # 1. Yhdistä WinAppDriveriin options = WindowsOptions() options.app = "Microsoft.Office.OUTLOOK_8wekyb3d8bbwe!microsoft.outlook" driver = webdriver.Remote( command_executor="http://127.0.0.1:4723", options=options ) # 2. Odota ja etsi Saapuneet-kansio nimellä inbox = driver.find_element( by="name", value="Inbox" ) inbox.click() # 3. Etsi ensimmäinen sähköposti listasta emails = driver.find_elements( by="class name", value="LeafRow" ) if emails: emails[0].click() print("[OK] Ensimmäinen sähköposti avattu") # 4. Lue sähköpostin sisältö body = driver.find_element( by="automation id", value="ReadingPaneContents" ) print(f"Sisältö: {body.text[:200]}") driver.quit() if __name__ == "__main__": test_outlook_winappdriver()
Huomaa ero: PyAutoGUI etsii kuvia pikselitasolla, Claude ymmärtää mitä ruudulla näkyy, ja WinAppDriver löytää elementtejä nimillä. Valinta riippuu käyttötapauksesta: PyAutoGUI on nopein aloittaa, Claude joustavin, ja WinAppDriver vakain.

Kokeile oikeasti: aja skriptit omalla koneellasi

Yllä oleva demo on selainsimulatio. Mukana tulee myös oikeat Python-skriptit jotka automatisoivat Laskin-sovellusta oikeasti. Alla ohjeet niiden ajamiseen.

Huom: Skriptit ottavat koneen hiiren ja näppäimistön haltuunsa ajon ajaksi. Turvamekanismi: siirrä hiiri ruudun nurkkaan, niin skripti pysähtyy välittömästi.

Esitiedot

  • Python 3.8+ asennettu
  • pip install pyautogui pillow (perusdemo)
  • pip install anthropic pyautogui pillow (Claude-demo)
  • macOS: Screen Recording -oikeus terminaalille (System Settings → Privacy → Screen Recording)

Vaihtoehto 1: PyAutoGUI Ei vaadi API-avainta

Avaa Laskin-sovelluksen, syöttää 42 + 58 näppäimistöllä, tarkistaa tuloksen ja ottaa kuvakaappauksen.

# 1. Asenna riippuvuudet pip install pyautogui pillow # 2. Aja demo (avaa Laskimen ja laskee 42+58) python scripts/demo_pyautogui.py

Vaihtoehto 2: Claude Computer Use AI-pohjainen

Sama testitapaus, mutta Claude AI analysoi ruudun visuaalisesti ja päättää itse mitä klikata. Ei tarvitse kovakoodata koordinaatteja tai referenssikuvia.

Avain ei tallennu mihinkään. Käytetään vain kopiointiin alla olevaan komentoon.
# 1. Asenna riippuvuudet pip install anthropic pyautogui pillow # 2. Aseta API-avain ja aja export ANTHROPIC_API_KEY="<syötä avain yllä>" python scripts/demo_claude_computeruse.py

Voit myös antaa oman tehtävän:

python scripts/demo_claude_computeruse.py \ --task "Avaa TextEdit, kirjoita 'Hello World' ja tallenna tiedosto nimellä testi.txt"

Mitä oikeat skriptit tekevät?

demo_pyautogui.py

  1. Avaa Laskin-sovelluksen (Calculator)
  2. Syöttää näppäimistöllä: 4, 2, +, 5, 8, Enter
  3. Kopioi tuloksen leikepöydälle (Cmd+C)
  4. Tarkistaa onko tulos 100
  5. Ottaa kuvakaappaukset todisteiksi
  6. Sulkee Laskimen

demo_claude_computeruse.py

  1. Avaa Laskin-sovelluksen
  2. Ottaa kuvakaappauksen ja lähettää Claudelle
  3. Claude analysoi ruudun ja päättää mitä klikata
  4. Suorittaa Clauden ehdottamat klikkaukset
  5. Toistaa: kuvakaappaus → analyysi → toiminto
  6. Claude raportoi tuloksen: PASS/FAIL

Testaajan työnkulku

Näin testaaja ottaa desktop-automaation käyttöön projektissa.

1

Asenna työkalut

# Perusasennus (PyAutoGUI-lähestymistapa) pip install pyautogui opencv-python pillow # TAI Claude Computer Use -lähestymistapa pip install anthropic pyautogui pillow

PyAutoGUI vaatii myös näytön käyttöoikeuden (macOS: System Preferences → Privacy → Screen Recording).

2

Ota referenssikuvat

Kuvakaappaa UI-elementit joita testi tarvitsee: napit, kentät, ikonit. Tallenna ne images/-kansioon. Kuvien tulee olla tarkkoja leikkauksia halutuista elementeistä.

Vinkki: Claude Computer Use -lähestymistavalla et tarvitse referenssikuvia ollenkaan! AI tunnistaa UI-elementit automaattisesti.
3

Kirjoita testiskripti

Määrittele testitapaus askeleina: mitä avataan, mitä klikataan, mitä syötetään, mitä tarkistetaan. Käytä wait_for_image()-funktioita odottamaan sovellusten latautumista.

4

Aja testit ja analysoi tulokset

Aja testiskripti ja tarkista tulokset. Kuvakaappaukset toimivat todisteina. Virhetilanteissa botti tallentaa kuvakaappauksen virheen hetkeltä diagnosointia varten.

# Aja testi python outlook_test.py # Tulosteet: [INFO] Käynnistetään Outlook... [OK] Outlook avautui [OK] Kirjautumistiedot syötetty [OK] Ensimmäinen sähköposti avattu [OK] Kuvakaappaus tallennettu
5

Integroi CI/CD-putkeen (valinnainen)

Desktop-testit voidaan ajaa CI/CD:ssä käyttäen virtuaalikoneita tai dedikoituja testikoneita. Huomaa: testikoneella täytyy olla aktiivinen näyttösessio (ei headless).

Tärkeää: Desktop-testejä EI voi ajaa headless-moodissa kuten web-testejä. Tarvitaan joko fyysinen näyttö, VNC-sessio, tai virtuaalinen framebuffer (Xvfb Linuxilla).

Käyttöönottotarkistuslista

  • Python 3.8+ asennettu
  • PyAutoGUI ja riippuvuudet asennettu
  • Näytön käyttöoikeudet myönnetty
  • Referenssikuvat otettu (jos PyAutoGUI)
  • Testiskripti kirjoitettu ja testattu manuaalisesti
  • Virheenkäsittely ja kuvakaappaukset virheen hetkellä
  • CI/CD-ympäristö konfiguroitu (näyttösessio)
  • Testidatan hallinta suunniteltu

Rajoitteet ja haasteet

Mitä on hyvä tietää ennen desktop-testausautomaation käyttöönottoa.

1. Resoluutio- ja teemariippuvuus

Kuvantunnistuspohjaiset testit ovat herkkiä näytön resoluutiolle, DPI-asetuksille ja käyttöjärjestelmän teemalle. Jos testikoneen näytön asetukset muuttuvat, referenssikuvat eivät enää täsmää.

Ratkaisu: Käytä alempaa confidence-tasoa (0.7-0.8), standardisoi testikone, tai siirry Claude Computer Use -pohjaiseen ratkaisuun joka sopeutuu muutoksiin.

2. Ei headless-tukea

Desktop-testejä ei voi ajaa ilman näyttöä. Tarvitaan aina aktiivinen näyttösessio - joko fyysinen, VNC, RDP tai virtuaalinen framebuffer.

3. Ajoitusongelmat

Sovellukset latautuvat eri nopeuksilla eri koneilla. Kiinteät sleep()-kutsut ovat hauraita. Parempi tapa: odota kunnes tietty UI-elementti ilmestyy (polling).

4. Ylläpitokustannukset

Referenssikuvat vanhenevat sovelluspäivitysten myötä. Jokainen Outlook-päivitys voi rikkoa testit. AI-pohjaiset ratkaisut (Claude Computer Use) ovat tässä robustimpia.

5. Rinnakkaistestaus

Kuvantunnistustestit hallitsevat koko näytön: hiiri ja näppäimistö ovat "varattuja". Rinnakkaisajoa varten tarvitaan erillisiä virtuaalikoneita tai -sessioita.

6. Tietoturva

Testiskriptit sisältävät usein kirjautumistietoja. Käytä aina ympäristömuuttujia tai salaisuudenhallintaa (vault) - älä kovakoodaa salasanoja skripteihin!

# Turvallinen tapa: käytä ympäristömuuttujia import os username = os.environ["TEST_USER"] password = os.environ["TEST_PASS"]

Milloin desktop-testausautomaatio kannattaa?

Kannattaa kun:

  • Toistuva manuaalinen testaus vie paljon aikaa
  • Testataan kriittisiä liiketoimintaprosesseja
  • Sovelluksella ei ole API:a tai web-käyttöliittymää
  • Regressiotestaus on tärkeää

Ei kannata kun:

  • Sovellus muuttuu jatkuvasti ja nopeasti
  • Testaus on kertaluontoista
  • Sama testaus voidaan tehdä API-tasolla
  • Ylläpitoresursseja ei ole saatavilla
